# Signature-proof examples

Concrete, verifiable examples of the proof a lender produces. When a lender signs the claim
form, the app shows two copyable boxes and `/submit` returns a receipt; these files are exactly
those shapes, for both signing paths the server accepts.

Everything here is generated by [`../scripts/generate-proof-examples.ts`](../scripts/generate-proof-examples.ts)
using the real signing/verification code in [`../src/utils.ts`](../src/utils.ts), so every
signature genuinely verifies. Regenerate after changing the signed structure:

```bash
npm run examples
```

> **These are synthetic.** The signer is a publicly-published test key (Hardhat/Anvil account
> `0xf39F…2266`), used only so the output is deterministic and obviously not a real lender. The
> market, borrower, amount and block are illustrative placeholders.

## The example claim

| Field           | Value                                        |
| --------------- | -------------------------------------------- |
| `network`       | `mainnet` (chainId `1`)                      |
| `market`        | `0x0000…00A1` (placeholder)                  |
| `penalizedDays` | `118`                                        |
| `amountOwedWei` | `250000000000` (250,000 of a 6-decimal asset)|
| `asOfBlock`     | `20812345`                                   |
| signer          | `0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266` |

## Files

| File | What it is |
| ---- | ---------- |
| [`eip712-signed-payload.json`](eip712-signed-payload.json) | The EIP-712 typed data the wallet is asked to sign (`domain` + `types` + `message`). Frontend "Signed payload" box. |
| [`eip712-proof.json`](eip712-proof.json) | The verification proof for the EIP-712 signature: recovered signer, the 32-byte typed-data hash, the signature, and the `/submit` receipt. Frontend "Signature & verification proof" box. |
| [`personal-sign-signed-payload.json`](personal-sign-signed-payload.json) | The plain UTF-8 message signed on the `personal_sign` fallback path (for wallets that can't do `signTypedData`). |
| [`personal-sign-proof.json`](personal-sign-proof.json) | The verification proof for the `personal_sign` signature. Note the `signature` carries the `personal_sign_` prefix the server keys on. |

## Two signing paths

- **EIP-712 (default).** Wallets sign structured typed data; the domain is bound to
  `chainId`, so a proof for one network can't be replayed on another. `verifySignature`
  (`src/utils.ts`) recovers the signer via `verifyTypedData`.
- **personal_sign (fallback).** Wallets that only support `personal_sign` sign the flat text in
  `personal-sign-signed-payload.json`. The client prefixes the signature with `personal_sign_`;
  the server strips it and recovers via `verifyMessage`.

Either way the claim commits to `{ network, market, penalizedDays, amountOwedWei, asOfBlock }`,
so the proof binds *which* market, *how much* was owed, and *at which block* — anyone can replay
`asOfBlock` on an archive node to confirm the figures.

## Smart-contract wallets (Safe / EIP-1271)

A Safe multisig has no private key to recover, so there is no static example here. The lender
signs the **same** typed data (or message); the server hashes it exactly as above — that hash is
the `typedDataHash` / `messageHash` in these proofs — and calls the wallet's
`isValidSignature(hash, signature)` (EIP-1271) on-chain instead of recovering an address. See
[`../../SAFE_MULTISIG.md`](../../SAFE_MULTISIG.md).

## Verifying these yourself

For the full flow the Wildcat Foundation runs on a real proof — signature **and** on-chain
replay at `asOfBlock` — see [`../../VERIFYING_PROOFS.md`](../../VERIFYING_PROOFS.md) (the
"Verify a proof" panel in the app, or `npm run verify`). Note these synthetic examples use a
placeholder market and a debug-faked amount, so they pass the signature check but not the
on-chain check — that's expected.

The committed files are re-checked by [`../test/examples.test.ts`](../test/examples.test.ts)
(`npm test`), which recovers the signer from each proof and confirms the committed hash. Ad hoc
signature check with ethers:

```js
const { verifyTypedData } = require('ethers');
const payload = require('./eip712-signed-payload.json');
const proof = require('./eip712-proof.json');
console.log(verifyTypedData(payload.domain, payload.types, payload.message, proof.signature));
// -> 0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266  (=== proof.signer)
```
